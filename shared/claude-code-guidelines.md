# Claude Code 運用ガイドライン（on the edge）

## コスト最適化ルール

### 絶対禁止
- `--resume` フラグの使用（キャッシュ完全無効化 → 20倍コスト）
- セッション中のMCPツール追加/削除（キャッシュ全体リビルド）
- セッション中のモデル切り替え（キャッシュはモデル別）

### 推奨プラクティス
- CLAUDE.mdは200行以下に保つ
- 詳細指示は .claude/rules/*.md にSkillsとして分離
- 5分以上の無操作を避ける（キャッシュTTL切れ）
- /compact 実行時は「変更ファイル一覧とテスト状態を保持」と指示
- /cost で定期的にコスト確認
- ccusage でセッション後にトークン消費分析

### CLAUDE.md更新ルール
- Claudeが間違えたら → その防止策をCLAUDE.mdに1行追記
- 週1回レビュー：不要な行を削除、重複を統合
- セッション中はCLAUDE.mdを変更しない（キャッシュ破壊防止）

## 全リポ共通ルール
- `git push` は松本の明示的な指示がある場合のみ実行
- エラー発生時は3回まで自力修正。解決できなければ状況報告して停止
- シークレット・APIキーは絶対にコミットしない
- .envファイルをgitに含めない
